# KT 수주전략 보고서 — 톤앤매너 표준 (Tone & Manner Standard)

본 문서는 KT 영업본부의 수주전략 보고서 공식 톤앤매너를 정의한다. 신규 RFP에 대응하는 수주전략 보고서를 작성할 때 본 표준을 따른다. 본 표준은 실제 위성영상 ISP 수주전략 보고서를 기반으로 추출되었다.

doc_type: win_strategy
customer: KT 영업본부 (내부 표준)
industry: 공공/SI/B2B 통합

---

## 1. 보고서 구조 (Document Architecture)

수주전략 보고서는 정형화된 3개 분석 슬라이드로 구성된다. 슬라이드 번호와 제목은 다음 순서를 따른다.

1. **사업 개요** — 사업내용 좌측 + 추진 방향 우측 2단 구성
2. **1. 고객/사업 현황 분석** — 과업범위 × 고객니즈/페인포인트 매트릭스
3. **2. 경쟁 현황 분석** — 과업범위 × (자사 vs 경쟁사) 좌우 비교
4. **3. 차별화 아이템 도출** — 과업범위 × 차별화 제안 매트릭스

이 4단 구조는 변경하지 않는다. 슬라이드 추가 시 4번 이후에 부속 자료로 배치한다.

## 2. 과업범위 분류 체계

모든 분석은 좌측 첫 컬럼 "과업범위" 기준으로 정렬한다. 표준 카테고리는 3종이다.

- **소프트웨어 개발** — 시스템 개발, 데이터 처리, UI/UX, 통합 솔루션
- **인프라 도입 및 환경 구축** — DC, 네트워크, 보안, VDI, 운영 관제, 백업, 기반시설
- **기타** — 업무 자동화, 미래 모델, 사업 관리, gray 영역, 컨설팅

모든 슬라이드에서 동일한 3분류를 유지하여 슬라이드 간 정합성을 확보한다.

## 3. 태깅 컨벤션 (Tagging Convention)

본문 모든 항목은 `[카테고리 태그] 본문 내용` 형식으로 표기한다. 대괄호 안에 핵심 키워드를 넣고, 콜론·하이픈 없이 본문이 바로 이어진다.

예시:
- `[데이터 표준화] 현재 각기 다른 표준의 위성영상 데이터 표준화 니즈`
- `[VDI 환경 구축] 서울-제주 실시간 판독 분석 가능한 VDI 환경 필요`
- `[DC 구축/운영] 국자원, 항우연 등 유사 사례 기반 안정적 구축 및 무중단 운영 방안 요청`
- `[gray영역 존재] ISP RFP 상 명확화 되지 않은 영역 다수`

태그는 명사형 키워드 1~2개로 구성한다. 동사형 표현(예: "구축하기")은 사용하지 않는다.

## 4. 상태 표시 라벨 (Status Badges)

분석 항목의 평가 결과는 색상 라벨로 표시한다.

- **빨간 박스 (RED)** — 부정적·약점·미흡 평가
  - `열위` — 경쟁사 대비 약함
  - `추가 발굴 필요` — 차별화 아이템이 부족하여 보강 필요
  - `리스크` — 사업 수행상 위험 요소
- **파란 박스 (BLUE)** — 긍정적·강점·우위 평가
  - `우위` — 경쟁사 대비 강함
  - `강점` — 자사 핵심 역량

색상 코드 (HEX):
- 빨강: #C01B2E
- 파랑: #1F4E9C

검정·회색 텍스트는 중립적 사실 진술에 사용한다.

## 5. 문장 종결 어미 (Sentence Endings)

수주전략 보고서는 객관적·추정적·요구사항 명시 어투를 혼용한다.

**고객의 의도 추정**: 
- "~원함" — 고객의 명시적 희망사항
- "~필요" — 객관적 필요성
- "~인식" — 고객 또는 시장의 인지 상태
- "~예상" — 향후 전개 추정

**자사 역량 진술**:
- "~가능" — 수행 가능성
- "~보유" — 자산·역량 명시
- "~제공" — 제안 가능 영역

**금지 어미**:
- "~하겠습니다", "~할 것입니다" 같은 다짐형 종결은 사용하지 않는다. 보고서는 사실·분석·제안이지 약속이 아니다.
- "~생각합니다", "~보입니다" 같은 주관성 종결도 피한다.

## 6. 정량 데이터 사용 규칙

추상적 표현 대신 구체 수치를 사용한다.

예시:
- `peta 단위 데이터` (X 대용량 데이터)
- `100GB급 서울-제주 연결 전용회선` (X 고속 전용회선)
- `4개년 대형사업` (X 장기 사업)
- `평가위원 8인` (X 평가위원단)
- `서울-제주간 3중화 가능 100기가급 백업망` (X 안정적 백업망)
- `최근접 지사` (X 가까운 지사)

수치가 없으면 "구체화 필요" 메모를 남긴다.

## 7. 레퍼런스 인용 패턴

자사 강점을 입증할 때 항상 실명 레퍼런스를 인용한다.

**공공 레퍼런스**: 국자원, 항우연, 국방부, NGA, NCG
**민간 레퍼런스**: 삼성전자(경쟁사 보유 시), 자사 보유 KT SAT
**기술 솔루션**: 엑사원(LG), 한컴 인터페이스, 컨텍, 세트렉아이

레퍼런스 명시 형식:
- `국자원 사업 등 공공 대형 레퍼런스` — 정확한 사업명·기관명
- `LL기반 공공 대형센터 구축 노하우` — Lessons Learned 약어 사용
- `KT+협력사 직원 현장 대응 지원 체계` — 컨소시엄 구성 명시

## 8. 경쟁사 분석 어조

경쟁사 약점은 사실 기반으로 서술하되, 비방·단정은 피한다.

권장 표현:
- "ISP 사업 수행 시 기간연장, 비용 증가 이슈로 인한 고객 불만 제기" — 사실 인용
- "~ 강조 예상" — 경쟁사 행보 추정
- "~ 추가 제안 예상" — 경쟁사 전략 예측

금지 표현:
- "경쟁사는 능력이 부족하다" (X 단정)
- "신뢰할 수 없다" (X 비방)

자사 약점도 동일한 객관성으로 인정한다. 예: `개발 관련 고객사 내부 상대적 인지도/이해도 부족 인식`

## 9. 차별화 아이템 도출 원칙

차별화 슬라이드는 다음 4유형 중 1개 이상을 명시한다.

1. **레퍼런스 이식형**: 유사 공공 대형 사업 노하우 적용 (예: 국자원 LL 이식)
2. **인프라 자산 활용형**: 자사 보유 네트워크·DC·인력 활용 (예: 서울-제주 100GB 전용회선)
3. **솔루션 차별형**: 검증된 자사 솔루션 적용 (예: 엔서블, AI ops)
4. **체계 차별형**: 운영 체계·SLA·대응 방식 차별화 (예: 최근접 지사 출동 체계)

차별화 항목이 부족하면 `추가 발굴 필요` 빨간 박스를 명시한다. 부족함을 숨기지 않는다.

## 10. gray 영역 처리

RFP 상 명확하지 않은 영역은 `gray영역 존재`로 명시하고, 기술협상 단계에서 논의할 사항으로 분류한다.

표현 예시:
- `[gray영역 존재] ISP RFP 상 명확화 되지 않은 영역 다수(향후 구축, 기술협상 시 논의 원함)`

gray 영역 발견은 약점이 아니라 기회로 본다. 사전 식별이 협상력의 원천이다.

## 11. 약어·전문용어 사용

업계 표준 약어는 풀네임 없이 사용한다.

허용 약어: VDI, DC, ISP, RFP, NGA, NCG, UPS, DCIM, PoC, LL, AI ops, SI, SLA
설명 필요 약어: KT SAT (KT 위성), 엑사원 (LG AI 모델), 엔서블 (오픈소스 자동화)

첫 등장 시 1회만 풀네임 병기:
- "VDI(Virtual Desktop Infrastructure)" — 최초 1회
- 이후 "VDI" 단독 사용

## 12. 평가 위원 분석 표기

슬라이드 하단에 평가위원 구성을 별표(*) 주석으로 명시한다.

형식: `* 평가 위원 현황: 고객사 내부 유관 부서 실무 담당자 N인으로 구성 예상, [구성 분야 명시]`

평가위원 분석은 영업 접점(S 점수)의 핵심 입력이며, 보고서마다 반드시 포함한다.

## 13. 보고서 작성 시 체크리스트

수주전략 보고서 작성 완료 전 다음을 확인한다.

- [ ] 4단 슬라이드 구조 준수 (개요 → 현황 → 경쟁 → 차별화)
- [ ] 과업범위 3분류 일관성 (모든 슬라이드 동일)
- [ ] 모든 본문 항목에 `[태그] 내용` 형식 적용
- [ ] 우위/열위/추가 발굴 필요 라벨 표기
- [ ] 정량 수치 ≥ 5개 명시
- [ ] 실명 레퍼런스 ≥ 3개 인용
- [ ] 자사 약점 1개 이상 객관 인정
- [ ] gray 영역 식별 결과 명시
- [ ] 평가위원 구성 별표 주석
- [ ] 약속·다짐형 종결 어미 미사용 확인

## 14. 본 표준의 적용 범위

본 톤앤매너는 다음 보고서에 적용한다.
- 사전 영업 단계의 수주전략 보고서
- RFP 분석 보고서
- 경쟁 현황 보고서
- 차별화 전략 보고서

다음에는 적용하지 않는다.
- 제안서 본문 (별도 톤앤매너)
- 임원 보고용 요약 (1장 압축 별도 양식)
- 사후 분석 보고서 (실주/수주 보고서는 별도 톤앤매너)
