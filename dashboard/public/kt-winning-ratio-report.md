# KT 수주전략 Winning Ratio 자동화 툴 — 데모 개요 보고서

> 작성일: 2026년 6월 | 목적: 수주 확률 자동화 플랫폼 컨셉 검증 데모

---

## 1. 왜 만들었는가

### Pain Point
KT 수주전략팀의 입찰 승률 판단은 현재 **담당자의 경험치와 직관**에 전적으로 의존한다.
- 판단 기준이 개인마다 달라 **일관성 부재**
- 복수 입찰 건의 우선순위 비교가 어렵고 **의사결정 지연**
- 승패 원인 분석이 축적되지 않아 **학습 효과 미흡**

### 목표
> 글로벌 정보 + 내부 DB + 집단지성을 결합해 **수주 성공 확률을 자동으로 산출**하고,  
> 확률을 높이기 위한 전략 아이디어를 즉시 제공하는 플랫폼 구축

- 의사결정 속도 향상 (분석 소요 시간 단축)
- 담당자 간 판단 일관성 확보
- 과거 수실주 데이터 기반 지속 학습

## 2. Polymarket에서 얻은 아이디어

### Polymarket이란
Polymarket은 **글로벌 예측 시장 플랫폼**이다. 불특정 다수가 "이 사건이 일어날 확률"에 베팅하고,
시장 가격이 곧 집단지성의 확률 추정값이 된다.

- 전문가 개인 예측보다 **집단 집계 예측이 더 정확**하다는 것이 검증됨 (prediction market 연구)
- 새로운 정보가 즉시 가격에 반영 → **실시간 확률 업데이트**

### 수주전략에 이식한 핵심 구조

| Polymarket 구조 | 수주 Winning Ratio 플랫폼 |
|---|---|
| 시장 참여자 베팅 | 수주팀·BD팀·임원 **전문가 투표** |
| 새 정보 → 가격 이동 | 시그널 카드 → **Bayesian 확률 업데이트** |
| 집단지성 확률 수렴 | 투표 누적 → **Posterior 확률 수렴** |
| 외부 뉴스·이벤트 반영 | 글로벌 뉴스 RAG → **드라이버 점수 자동 산출** |

**차이점**: 폴리마켓은 금전 베팅 기반 / 이 플랫폼은 **도메인 드라이버(5축) 기반 구조화 확률**

## 3. 현재 데모 구현 방식 — 글로벌 뉴스 RAG로 대체

### 배경
KT 수실주 DB(경쟁사 정보, 수실주 보고서, 발주사 정보 등)가 현재 미확보 상태이므로,
**글로벌 뉴스 데이터**를 대체 신호로 활용해 알고리즘 전체를 구현·검증했다.

### 구현 흐름

```
주제 입력 (예: "이란 전쟁 종결 가능성")
    ↓
글로벌 뉴스 RAG 검색
(Foreign Affairs · The Economist · Financial Times 등)
    ↓
GPT OSS (o4-mini) — 5개 드라이버 자동 생성 + 점수 산출
    ↓
Bayesian Prior 확률 계산
    ↓
시그널 카드 생성 (4~7장)  ←→  전문가 투표 누적
    ↓
Posterior 확률 수렴 + 전략 아이디어 생성
    ↓
PDF 리포트 발행
```

### 알고리즘 핵심

- **5축 드라이버 프레임워크**: 주제별 자동 생성 (예: 외교·군사·경제·국내정치·국제압력)
  - 각 드라이버 0~10점 → 기여도 계산 (invert 플래그: 높을수록 확률 하락하는 축 처리)
- **Bayesian Ensemble**: `Posterior = Prior × PRIOR_STRENGTH + Σ(투표 가중 델타)` → 확률 수렴
- **시그널 카드**: 뉴스 이벤트를 구조화 카드로 변환, 각 카드가 드라이버 점수에 ±영향
- **현장 의견 반영**: QR 투표 페이지에서 자유 텍스트 입력 → GPT가 시그널로 변환 → 즉시 확률 반영

### 검증 결과
데이터 없이도 **알고리즘 구조·UX·투표 메커니즘·리포트 생성 전체 동작 확인 완료**

## 4. 실 DB 연동 시 그대로 적용 가능한 이유

알고리즘과 플랫폼 구조는 변경 없이, **데이터 소스만 교체**하면 수주전략에 직접 적용 가능하다.

| 구성 요소 | 현재 데모 (뉴스 RAG) | 실 DB 연동 후 |
|---|---|---|
| **데이터 소스** | 글로벌 뉴스 기사 | 수실주 보고서 · 경쟁사 DB · 발주사 정보 |
| **드라이버 5축** | 지정학 축 (자동 생성) | 수주전략 축: 기술력 · 가격경쟁력 · 관계 · 레퍼런스 · 리스크 |
| **시그널 카드** | 뉴스 이벤트 카드 | 발주사 동향 · 입찰 변수 · 경쟁사 움직임 |
| **투표자** | 데모 참여자 | 수주팀 · BD팀 · 임원 |
| **확률 의미** | 지정학 사건 발생 가능성 | **수주 성공 확률** |
| **전략 출력** | 국제 정세 대응 전략 | 수주 확률 제고 전략 (액션 플랜) |

→ Bayesian 업데이트 · RAG 파이프라인 · 투표 메커니즘 · 리포트 생성 **코드 재사용률 90%+**

## 5. 다음 단계 제안

**Phase 1 — 데이터 파이프라인 구축** (1~2개월)
1. KT 수실주 DB 연결: 과거 5년 입찰 결과 → 드라이버 가중치 학습
2. 경쟁사 정보 자동 수집: 공시 데이터 · 뉴스 모니터링 → 시그널 카드 자동 생성
3. 발주사 성향 프로파일링: 담당자 · 부서별 의사결정 패턴 모델링

**Phase 2 — 수주팀 실 운영 파일럿** (2~3개월)
4. 수주팀 · BD팀 계정 등록 → 투표 권한 부여 (역할별 가중치 설정)
5. 진행 중인 입찰 건 실시간 입력 → 확률 추이 모니터링
6. 승패 결과 피드백 루프 → 드라이버 가중치 자동 보정

**Phase 3 — 고도화**
7. 경쟁사 A/B 시나리오 시뮬레이터 (What-if 분석)
8. 임원 대시보드: 전체 포트폴리오 수주 확률 히트맵
9. 모바일 QR 투표 → 현장 인텔리전스 실시간 수집
